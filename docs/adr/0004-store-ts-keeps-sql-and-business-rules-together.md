# store.ts keeps SQL and business rules together (for now)

`src/lib/store.ts` is ~960 lines spanning SQL, schema, migration, row mappers, link revalidation, and stats aggregation. The natural deepening is to extract `revalidateLinks` into a pure `processRevalidationBatch(rows) → Patch[]` and apply patches separately — turning the implicit state machine into a testable function. **Deferred** because the lost-update race bugs that would have justified the split are already fixed at the SQL layer (narrow per-id `UPDATE` inside `withLock` for both `revalidateLinks` and `scanPortals`). Re-touching the file now would churn code that just stabilized. Reopen when revalidation grows another state (e.g., a "needs human review" intermediate archive state) or when a non-LinkedIn source-specific revalidator joins (combine with [[0001-posting-health-stays-three-modules]]).
